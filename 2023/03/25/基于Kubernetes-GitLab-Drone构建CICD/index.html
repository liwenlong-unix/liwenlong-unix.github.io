基于Kubernetes+GitLab+Drone构建CICD

1.案例目标

（1）掌握Drone的使用和管理。

（2）掌握Gitlab的使用和管理。

（3）了解CICD的配置步骤和方法。

2.节点规划

IP 主机名 节点
172.129.101.43 k8s-master-node1 master节点、Harbor仓库节点
172.129.101.74 k8s-worker-node1 node节点

3.案例实施

3.1部署gitlab

解压软件包并导入镜像

部署Gitlab

GitLab启动较慢，等待5-10分钟左右，在浏览器上通过http://master_IP:30888访问GitLab

登录Gitlab,用户名为root，密码为P@sswOrd。

点击右上角头像，然后点击”Edit profile“进入User Settings界面

点击左侧导航栏“Applications”，创建一个GitLab OAuth应用程序，名字为drone，Redirect URL为Drone登录地址，并勾选api和read_user。

配置完成后保存。

登录admin管理员页面（http://master_IP:30888/admin），依次点击“Settings”→“Network”→“Outbound requests”，勾选“Allow requests to the local network from web hooks and services”。

点击“Save changes”保存。

3.2导入项目到Gitlab

登录GitLab首页,点击“New project”

选择“Import project”,点击“GitLab export”，然后在“Project name”和“Project slug”处输入“demo-2048”，点击“选择文件”，选择提供的软件包demo-2048.tar.gz。

点击“Import project”导入项目
